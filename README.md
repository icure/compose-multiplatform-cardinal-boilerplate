Kotlin Compose Multiplatform boilerplate app for Cardinal on Android, iOS, or Desktop (JVM).

This application already includes the basic registration/login flow using an authentication process
and example usage of the SDK.

## Configuration

Before running the application, make sure to configure the following settings:

- EXTERNAL_SERVICES_SPEC_ID id for the configuration of email/sms/captcha services for your environment, obtained from the cockpit. 
- PROCESS_ID: id of the process for login/registration created from the cockpit.
- APPLICATION_ID: the identifier for your cardinal solution.
- PRODUCT_BUNDLE_IDENTIFIER: the identifier of your iOS and android applications.
